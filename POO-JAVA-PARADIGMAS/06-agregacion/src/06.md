# 06 - Agregación

Relación "tiene un" débil: las partes pueden existir sin el todo.

## Archivos típicos
- `Jugador.java` → nombre, posición.
- `Equipo.java` → lista de Jugadores (recibida por parámetro).
- `Main.java` → Crea jugadores y luego el equipo.

## ¿Qué se aprende?
- El todo no es dueño de las partes.
- Las partes se crean fuera.
